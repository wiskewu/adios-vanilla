双仓库GithubActions部署博客

Note

本文内容基于【梦爱吃鱼】使用 Github Actions 实现博客自动化部署

前提准备

  • Github账号x1
  • 电脑x1
  • 勤劳的小手x1

本教程是通过Github Actions实现持续集成和持续部署（CI/CD），自动化生成静态文件并推送到目标仓库。
本文需要两个仓库，一个私有一个公开：

  • ssg-generator: 私有仓库，存放生成静态博客网站的源代码
  • blog-site: 公开仓库，存放静态博客内容的最终产物（HTML、CSS、JS），产物由ssg-generator生成

上面两个仓库名仅作示例和演示，具体以你实际项目为准。

创建工作流

什么是Github Actions

GitHub Actions 是一个持续集成和持续部署（CI/CD）平台，可以在 GitHub 仓库中配置工作流。当有新的推送到达 GitHub 仓库时，GitHub Actions 会自动触发预定义的工作流。

  • 方法一：在ssg-generator代码项目根目录下创建.github/workflow目录，然后在这个目录中创建一个新的YAML文件，例如deploy.yaml，填入工作流配置；
  • 方法二：在ssg-generator仓库界面依次点击Actions->set up a workflow yourself，填入工作流配置（注意链接中的用户名和项目名替换为你的实际用户名和项目名，否则是无效的链接地址）

无论采用上述哪个方法，填写的内容都是下面的配置：

yaml 
name: Github Action自动构建

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    env:
      TZ: Asia/Shanghai
    steps:
    - name: 检查源仓库分支
      uses: actions/checkout@v3
      with:
        ref: main
        fetch-depth: 0

    - name: 安装 Node.js
      uses: actions/setup-node@v3
      with:
        node-version: "20.x"

    - name: 安装 Yarn
      run: |
        npm install -g yarn

    - name: 缓存 node 模块
      id: cache-node-modules
      uses: actions/cache@v3
      with:
        path: node_modules
        key: ${{ runner.os }}-node-modules-${{ hashFiles('**/yarn.lock') }}
        restore-keys: |
          ${{ runner.os }}-node-modules-

    - name: 安装依赖
      if: steps.cache-node-modules.outputs.cache-hit != 'true'
      run: |
        yarn

    - name: 生成静态
      run: |
        yarn build

    - name: 部署推送到目标仓库
      uses: peaceiris/actions-gh-pages@v3
      with:
        personal_token: ${{ secrets.PAT }}
        publish_dir: ./public
        external_repository: wiskewu/blog-site
        user_name: "github-actions[bot]"
        user_email: "github-actions[bot]@users.noreply.github.com"
        full_commit_message: "通过 Github Actions 部署静态博客站点构建"
        publish_branch: main

上述高亮区域，需要根据自己的仓库信息进行更改。
上述工作流做的事情是：

  • 当ssg-generator的main分支收到新的推送时，就会启动工作流；
  • 首先安装yarn依赖管理工具（根据项目选择你自己的依赖管理工具npm/pnpm/yarn...）
  • 然后根据项目配置安装所需依赖yarn install（同上，选择yarn就执行yarn）
  • 然后执行yarn run build开始构建生成静态博客（同上，根据自己项目的打包命令配置）
  • 然后将生成的产物目录public推送至blog-site仓库的main分支（此处也是根据项目选择）
  • 推送的commit信息就是full_commit_message中配置的内容

配置访问令牌

想要在ssg-generator仓库的工作流中推送代码至blog-site，需要配置个人访问令牌来获得足够的权限。

生成访问令牌

  1. 进入Settings -> Developer Settings -> Personal access tokens -> Tokens（Classic）页面
  2. 点击Generate new Token，输入令牌的描述，选择需要的权限（repo权限即可）
  3. 点击Generate token生成令牌
  4. 复制生成的令牌

添加到Secrets

  1. 进入ssg-generator仓库
  2. 进入Settings -> Secrets and variables -> Actions（注意链接中的用户名和项目名替换为你的实际用户名和项目名，否则是无效的链接地址）
  3. 点击New repository secret，输入PAT，粘贴你的个人访问令牌到Secret *字段，然后点击Add secret

触发工作流

修改ssg-generator仓库的内容并提交commit，触发工作流，首次可能比较慢，因为工作流配置了缓存 Node.js 依赖。
